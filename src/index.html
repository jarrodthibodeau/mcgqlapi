Metacritic GraphQL API

An unofficial GraphQL API for Metacritic! THIS IS AN ALPHA, PLEASE EXPECT BUGS ALONG THE WAY The API as of right now currently supports

  • Getting any game(s)
  • Getting any movie(s)
  • Getting any TV season(s)
  • Getting any album(s)

In the future, the API will support

  • Person
    • Any information regarding a person (actor, musician, etc)'s projects
  • Company
    • Any information regarding a company's projects
  • Reviews
    • Being able to access the reviews on a particular product

Endpoint

All requests should go to the /graphql

Schema

The schema is outlined as such.


  type Query {
    game(input: Game!): GameInfo
    games(input: [Game!]): [GameInfo]
    album(input: Album!): AlbumInfo
    albums(input: [Album!]): [AlbumInfo]
    movie(input: Movie!): MovieInfo
    movies(input: [Movie!]): [MovieInfo]
    tvshow(input: TVShow!): TVShowInfo
    tvshows(input: [TVShow!]): [TVShowInfo]
  }

  type GameInfo {
    title: String
    platform: String
    criticScore: Int
    developer: String
    publisher: String
    genres: [String]
    numOfCriticReviews: Int
    numOfPositiveCriticReviews: Int
    numOfMixedCriticReviews: Int
    numOfNegativeCriticReviews: Int
  }

  input Game {
    title: String!
    platform: String!
  }

  type AlbumInfo {
    album: String
    artist: String
    criticScore: Int
    publisher: String
    genres: [String]
    numOfCriticReviews: Int
    numOfPositiveCriticReviews: Int
    numOfMixedCriticReviews: Int
    numOfNegativeCriticReviews: Int
  }

  input Album {
    artist: String!
    album: String!
  }

  type MovieInfo {
    title: String
    criticScore: Int
    year: String
    director: [String]
    genres: [String]
    cast: [String]
    rating: String
    runtime: String
    summary: String
    numOfCriticReviews: Int
    numOfPositiveCriticReviews: Int
    numOfMixedCriticReviews: Int
    numOfNegativeCriticReviews: Int
  }

  input Movie {
    title: String!,
    year: String
  }

  type TVShowInfo {
    title: String
    season: String
    criticScore: Int
    genres: [String]
    summary: String
    numOfCriticReviews: Int
    numOfPositiveCriticReviews: Int
    numOfMixedCriticReviews: Int
    numOfNegativeCriticReviews: Int
  }

  input TVShow {
    title: String!
    season: String
  }

Example Request

An example query would look like so

The Query

query ($input: Game!) {
	game(input: $input) {
		title
		platform
		criticScore
		developer
		publisher
		genres
		numOfCriticReviews
		numOfPositiveCriticReviews
		numOfMixedCriticReviews
		numOfNegativeCriticReviews
	}
}

The Variables

{
	"input": {
		"title": "Resident Evil 2",
		"platform": "PlayStation 4"
	}
}

The Response

{
  "data": {
    "game": {
      "title": "Resident Evil 2",
      "platform": "PlayStation 4",
      "criticScore": 91,
      "developer": "Capcom R&D Division 1",
      "publisher": "Capcom",
      "genres": [
        "Action Adventure",
        "Survival"
      ],
      "numOfCriticReviews": 85,
      "numOfPositiveCriticReviews": 84,
      "numOfMixedCriticReviews": 1,
      "numOfNegativeCriticReviews": 0
    }
  }
}

Caveats

TV Shows

If you do not provide a season for the TV show, you will get an overview of the series as a whole. This includes an overall series score and reviews for all the seasons.

Movies

As there are plenty of remakes in the movie space. Any newer versions of a movie should include the year of release to ensure the correct movie is obtained

Example:

Little Women will return the 1994 version

Little Women 2019 will return the 2019 version
